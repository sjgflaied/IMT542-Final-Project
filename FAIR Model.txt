Mental Health Crisis Early-Warning System (MHCEWS): The MHCEWS restructures the SAMHSA NSDUH State Prevalence Tables into state-level mental health risk records, each uniquely identified by a composite key of state_abbr (USPS two-letter code) + survey_year (NSDUH release year), and described through a four-layer metadata structure (Identity, Signal, Prediction, Action) plus a provenance block, providing both human-readable labels and machine-readable typed values so that downstream systems can interpret the data without the external NSDUH PDF codebook ('F'). Records are accessible via a public REST API over HTTPS using standard, open HTTP protocols that are free and universally implementable; because the underlying NSDUH data is U.S. Government public-domain and contains only state-level aggregates with no individual-level information, all risk outputs are openly readable with no authentication required, and the API exposes only GET routes so there is no path through which a caller can mutate state ('A').
All data fields follow snake_case conventions aligned with JSON-LD practice — any_mi_pct, smi_pct, mde_pct, tx_rate_pct, care_gap, risk_score, risk_tier, surge_probable — replacing the original NSDUH CSV's codebook-dependent column abbreviations (AMIPY, SMIPY, MDEPY, TXRECMI) and untyped string values with typed floats and a documented enum, while geographic identifiers use the standardized USPS state abbreviation vocabulary so any client adopting these widely used standards can consume the payload without preprocessing ('I'). Each record includes a provenance block capturing data_source (e.g. samhsa_nsduh_2023), model_version (semver, bumped on every retrain so old versions remain reproducible), and survey_year, enabling full traceability of how every risk_score is generated back to the upstream NSDUH release that produced it ('R'). Public outputs are released under a Creative Commons Attribution 4.0 license; the underlying SAMHSA NSDUH State Prevalence Tables are U.S. Government public-domain — ensuring clarity in reuse conditions and supporting transparency and reproducibility ('R').
